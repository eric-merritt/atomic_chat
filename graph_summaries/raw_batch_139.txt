function::services/llama.py::loaded_model_id | This function retrieves the currently loaded model ID from the Llama server, using a cache to avoid frequent network requests. It handles exceptions by returning None and updates the cache with the new value and expiration time.
function::services/llama.py::llama_is_healthy | This function checks the health of the Llama server by sending a GET request to the /health endpoint. It returns True if the status code is 200, otherwise it returns False.
function::services/llama.py::_read_pid | This function reads the process ID from a file and verifies if the process is still running. It returns the PID if the process exists, or None if the file is missing or the process is not found.
function::services/llama.py::kill_llama_server | This function terminates the Llama server process by sending SIGTERM and waiting for it to exit, escalating to SIGKILL if necessary. It also invalidates the model cache and removes the PID file.
function::services/llama.py::spawn_llama_server | This function starts the Llama server process with the specified model configuration and arguments. It waits for the server to become healthy and confirm the model is loaded, returning True on success or False on timeout.
