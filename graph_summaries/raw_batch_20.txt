function::auth/credentials.py::_encrypt | This function encrypts the provided data bytes using AES-GCM if the cryptography library is available, otherwise it falls back to an insecure XOR encryption. It returns the nonce concatenated with the ciphertext for AES-GCM, or a prefixed XOR-encrypted byte string for the fallback method.
function::auth/credentials.py::_decrypt | This function decrypts the provided blob by checking for an XOR prefix or using AES-GCM decryption based on the available libraries. It raises a RuntimeError if the blob was encrypted with the cryptography library but that library is not currently installed.
function::auth/credentials.py::load_credentials | This function loads and decrypts the credentials file from disk, returning an empty dictionary if the file does not exist or if decryption fails. It handles exceptions gracefully by printing a warning to stderr and returning an empty dictionary in case of errors.
function::auth/credentials.py::save_credentials | This function serializes the credentials dictionary to JSON, encrypts it, and writes the resulting bytes to the credentials file. It also sets the file permissions to 0o600 to ensure only the owner can read or write to it.
function::auth/credentials.py::add_credential | This function loads the existing credentials, updates or adds a new entry with the provided alias, URL, and type, and then saves the updated credentials back to the store. It supports various credential types such as basic, api_key, cookie, and bearer.
