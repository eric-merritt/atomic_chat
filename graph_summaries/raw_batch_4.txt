function::atomic_client/agent.py::_check_session | This function validates a session ID by sending a GET request to the authentication endpoint. It returns True if the response is successful, otherwise it returns False.
function::atomic_client/agent.py::_browser_auth | This function initiates a browser-based authentication flow by generating a token and opening a URL in the user's default browser. It polls the server for approval status and returns the session ID if approved, or None if denied or timed out.
function::atomic_client/agent.py::ensure_session | This function ensures a valid session ID exists by checking stored credentials and validating them. If no valid session is found, it triggers browser authentication and saves the new session ID.
function::atomic_client/agent.py::do_challenge_response | This function performs a cryptographic challenge-response authentication over a WebSocket connection. It fetches the server's public key, encrypts a received nonce, sends the response, and verifies if authentication was successful.
function::atomic_client/agent.py::_ensure_bash_disclaimer | This function checks if a disclaimer file exists and prompts the user to accept terms if it does not. It exits the program if the user declines and creates the file if they accept.
